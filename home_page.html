Documentation Sync Agent (EPIC 2)

The Documentation Sync Agent is an event-driven automation built using Flowise (Flogentic-style orchestration). It listens to GitHub repository events, detects documentation-related changes, generates updated Low Level Design (LLD) content using AI, and automatically synchronizes the master project documentation.

Purpose

The primary goal of this agent is to ensure that technical documentation remains continuously aligned with the source repository. Manual updates to Low Level Design documents are eliminated by automatically regenerating and updating the LLD section whenever documentation changes occur.

Key Capabilities

GitHub Event Trigger

Listens to GitHub webhook events such as commit pushes, pull requests, or workflow runs to initiate the documentation synchronization workflow.

Markdown Change Detection

Detects changes only in markdown-based documentation files and prepares normalized content for AI-driven analysis.

AI-Driven LLD Generation

Uses a Flowise-integrated Large Language Model (LLM) to generate updated Low Level Design content along with a clear summary of what changed and why.

Automated Documentation Update

Automatically replaces the LLD section inside the master project documentation file while keeping all other sections unchanged.

Version-Controlled Updates

Commits the updated documentation back to the GitHub repository, ensuring traceability, auditability, and rollback support.

Governed Publishing

Documentation updates follow internal governance rules. Access-controlled publishing and notifications are planned as future enhancements.

What Exactly Happens When a Change Is Pushed?

  • GitHub triggers a webhook event on commit or pull request.
  • The backend validates the event and extracts repository metadata.
  • Only markdown documentation files are identified for processing.
  • Updated markdown content is fetched, cleaned, and normalized.
  • The content is forwarded to an AI agent via Flowise.
  • The AI regenerates the updated Low Level Design (LLD).
  • The LLD section in the master documentation is automatically replaced.
  • The updated document is committed back to the repository.

System Components

Backend Components

  • Webhook Receiver – Receives and validates GitHub events.
  • Markdown Change Detector – Identifies modified documentation files.
  • Raw Content Fetcher – Retrieves markdown content from GitHub.
  • Flowise Forwarder – Sends structured inputs to the AI agent.
  • Diff Analyzer – Determines documentation impact scope.
  • Logging Layer – Captures execution flow and errors.

Flowise Agent Components

  • Custom Function Node (Extractor) – Parses GitHub payloads and prepares markdown context.
  • LLM Node – Generates updated LLD content and change summary.
  • Custom Function Node (Updater) – Updates the master documentation and commits changes.

Change Impact Summary

Any change to markdown-based documentation files triggers regeneration of the Low Level Design section. Non-documentation code changes do not affect the documentation unless explicitly enabled in future enhancements.

The Documentation Sync Agent updates only the LLD section marked for automation. All other sections of the project documentation remain unchanged to preserve stability and review traceability.
